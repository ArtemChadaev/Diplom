# Алгоритмы работы системы ERP (Фармацевтика)

Данный документ описывает общую логику работы системы и детальные алгоритмы ключевых бизнес-процессов.

---

## 1. Общий алгоритм работы системы (High-Level)

Система построена на принципе сквозного контроля жизненного цикла лекарственных средств (ЛС) от момента поступления на склад до отгрузки конечному потребителю (аптеке).

### Основные этапы:
1.  **Приемка (Inbound):**
    *   Поступление товара по накладной (ТТН).
    *   Автоматическое создание серий (batches) в статусе `quarantine`.
    *   Размещение по зонам хранения в зависимости от условий (холодовая цепь, НС/ПВ, огнеопасные).
2.  **Карантин и Контроль качества:**
    *   Проверка Уполномоченным лицом (QP).
    *   Подписание протокола осмотра. Перевод серий в статус `available` (разрешено к продаже) или `rejected` (брак).
3.  **Хранение и Мониторинг:**
    *   Ежедневный (утро/вечер) замер температуры и влажности в каждой зоне.
    *   Блокировка серий при получении STOP-сигналов от регулятора (Росздравнадзор).
    *   Периодические слепые инвентаризации.
4.  **Отгрузка (Orders):**
    *   Формирование заказа аптекой.
    *   Автоматический подбор серий по принципу **FEFO**.
    *   Сборка, проверка и печать реестра качества.

## 2. Алгоритм списания препаратов по принципу FEFO

**FEFO (First Expired, First Out)** — приоритет отгрузки препаратов, у которых срок годности истекает раньше.

### Входные данные:
*   `product_id` (ID препарата)
*   `requested_qty` (Требуемое количество)
*   `order_type` (Тип заказа: Regular или CITO)

### Схема алгоритма:
1.  **Поиск доступных остатков:**
    *   Запрос к таблице `batches`.
    *   Фильтр: `product_id = P`, `status = 'available'`, `quantity > 0`.
2.  **Сортировка:**
    *   Сортировка списка по `expiry_date` (дата истечения) **ASC** (по возрастанию).
    *   Если даты совпадают — сортировка по `manufacture_date` ASC.
3.  **Итерационный подбор:**
    *   Создать пустой список `selected_batches`.
    *   Переменная `remaining = requested_qty`.
    *   **Цикл** по отсортированным сериям:
        *   Если `batch.quantity >= remaining`:
            *   Добавить в `selected_batches`: `{batch_id, qty: remaining}`.
            *   `remaining = 0`.
            *   Прервать цикл.
        *   Иначе:
            *   Добавить в `selected_batches`: `{batch_id, qty: batch.quantity}`.
            *   `remaining -= batch.quantity`.
4.  **Проверка результата:**
    *   Если `remaining > 0`: выбросить ошибку "Недостаточно товара на складе".
5.  **Блокировка подбора (VEN/MOS анализ):**
    *   *Условие:* Если после подбора остаток препарата в системе падает ниже заданного порога (напр. 60% от среднего месячного оборота — МОС).
    *   *Действие:* Маркировать позицию заказа статусом `mos_blocked`, если тип заказа `regular`. Для `cito` заказов блокировка игнорируется.
6.  **Выход:** Список пар `(batch_id, quantity)` для резервирования.

## 3. Алгоритм блокировки списания по VEN-анализу

**VEN-анализ** классифицирует препараты на:
*   **V (Vital)** — жизненно важные (постоянное наличие обязательно).
*   **E (Essential)** — важные.
*   **N (Non-essential)** — второстепенные.

### Логика блокировки:
1.  **Определение категории:** При попытке отгрузки система проверяет флаг `is_jnvlp` (ЖНВЛП) или специфический атрибут категории VEN в карточке `Product`.
2.  **Проверка лимита (Safety Stock):**
    *   Для категории **V**: Если текущий остаток на складе < `safety_stock_limit`, отгрузка по обычным заказам (`order_type = 'regular'`) **блокируется**. 
    *   Разрешается отгрузка только для экстренных нужд (`order_type = 'cito'`).
3.  **Ветвление:**
    *   `IF product_category == 'V' AND total_stock < min_threshold`:
        *   `IF order_type != 'cito'`:
            *   Вернуть статус: `ERROR_V_CATEGORY_RESERVE_ONLY`.
            *   Уведомление Менеджеру закупки.
    *   `ELSE`: Продолжить процесс FEFO.

## 4. Алгоритм расчета «Точки перезаказа» (Reorder Point, ROP)

Алгоритм помогает автоматически формировать заявки поставщикам.

### Формула:
`ROP = (S * L) + SS`

Где:
*   **S (Average Daily Usage):** Среднесуточное потребление за последние 30-90 дней.
*   **L (Lead Time):** Время выполнения заказа поставщиком (в днях).
*   **SS (Safety Stock):** Страховой запас (напр. 10% от потребности).

### Шаги расчета:
1.  **Сбор статистики:** Анализ таблицы `order_items` за период.
2.  **Оценка остатков:** Сумма `quantity` по всем сериям `available`.
3.  **Сравнение:**
    *   Если `Current Stock <= ROP`:
        *   Статус товара: **Needs Reorder**.
        *   Автоматическая генерация черновика заказа поставщику с `quantity = Max_Stock - Current_Stock`.
