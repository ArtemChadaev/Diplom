# Frontend — Обзор и Стандарты Разработки

← [Назад к главному README](../../README.md) | [Конвенции →](./conventions.md) | [Спецификации UI форм →](./forms/) | [Логика форм →](./forms-logic.md) | [Архитектура FSD →](./fsd.md)

> ⛔ **Правило веток:** Весь код фронтенда **должен** разрабатываться только в ветке `develop-frontend`.
> Прямые коммиты в `main` или `develop-backend` строго запрещены.
> Слияние в `main` выполняется исключительно через автоматический воркфлоу [`/merge-all`](../other/git-workflow.md).

Фронтенд ERP-системы фармацевтического склада построен на стеке **React 19 + Vite + TypeScript** с использованием методологии **Feature-Sliced Design (FSD)**.

---

## 🛠️ Стек технологий

| Технология | Версия | Роль |
|-----------|---------|------|
| **React** | 19 | UI библиотека |
| **Vite** | Последняя | Сборщик и dev-сервер |
| **TypeScript** | Strict mode | Язык разработки |
| **Feature-Sliced Design** | FSD | Архитектурная методология |
| **React Router** | v7 | Маршрутизация |
| **Tailwind CSS** | v4 | Стилизация |
| **shadcn/ui** | Последняя | Компонентная база |
| **lucide-react** | Последняя | Иконки |
| **Zod** | v4 | Валидация схем и данных |
| **date-fns** | v4 | Работа с датами |
| **Axios** | Последняя | Сетевой API-клиент (интерцепторы, Bearer, silent-refresh) |
| **Zustand** | Последняя | Стейт-менеджер сессии пользователя (с localStorage-персистентностью) |

## 🚀 Локальная разработка

```bash
cd frontend
pnpm install
pnpm dev    # http://localhost:3000
```

Запуск через Docker Compose из корня монорепозитория:

```bash
docker compose --profile all up -d
```

## 🎨 index.css — Правило неизменяемости

> ⛔ **Файл `src/index.css` сгенерирован через shadcn CLI (пресет `b2p9XgkXg`, тема `radix-lyra`). Строго запрещено менять существующие токены или добавлять новые без согласования.**

Правила стилизации:
1. **Используйте только семантические переменные** — `bg-primary`, `text-foreground`, `border-border`, `rounded-xl`, а также специализированные статусные цвета: `bg-info`, `bg-warning`, `bg-notification`.
2. **Запрет сырых утилит Tailwind** — никогда не используйте `bg-gray-800`, `text-red-500` и аналогичные классы.

```
✅  bg-primary          ❌  bg-gray-800
✅  text-destructive    ❌  text-red-500
✅  bg-info             ❌  bg-blue-500
✅  bg-warning          ❌  bg-amber-500
✅  bg-notification     ❌  bg-purple-500
```

→ Полный список переменных: [`conventions.md — Стилизация`](./conventions.md#стилизация)

## 📁 Структура проекта (FSD)

Проект строго следует методологии **Feature-Sliced Design**:

- **`app/`** — инициализация, провайдеры, роутер, глобальные стили.
- **`pages/`** — страницы (композиция виджетов и фич).
- **`widgets/`** — крупные самостоятельные блоки (Header, Footer, Layout).
- **`features/`** — бизнес-логика и действия пользователя (формы, обработчики).
- **`entities/`** — бизнес-сущности (User, Product, Order).
- **`shared/`** — переиспользуемый код (UI-кит shadcn, utils, api-клиент).

→ Подробнее об архитектуре и правилах колокации: [`fsd.md`](./fsd.md)
