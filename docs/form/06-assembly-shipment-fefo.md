# Форма 4.1–4.2: Сборка заказа и Контроль отгрузки (FEFO)

## Требования (пресловой-чеклист)

- [ ] Таблица `orders`: `id`, `type: enum(regular, cito)`, `status`, `destination` (аптека), `created_at`, `assembled_by`
- [ ] Таблица `order_items`: `id`, `order_id`, `product_id`, `requested_qty`, `batch_id` (автовыбор по FEFO), `assembled_qty`, `status`
- [ ] Таблица `batches` содержит `expiry_date` — на сервере реализован FEFO-алгоритм выбора серии
- [ ] Бэкенд: `POST /api/v1/orders` — создание заказа
- [ ] Бэкенд: `GET /api/v1/orders` — список заказов (с фильтрами: статус, тип, дата)
- [ ] Бэкенд: `GET /api/v1/orders/:id` — детали заказа с FEFO-рекомендациями
- [ ] Бэкенд: `POST /api/v1/orders/:id/confirm-assembly` — подтверждение сборки
- [ ] Бэкенд: `POST /api/v1/orders/:id/ship` — (admin/qp/warehouse_manager) подтверждение отгрузки + генерация ТТН
- [ ] Бэкенд: `GET /api/v1/orders/:id/quality-registry` — реестр сертификатов к ТТН
- [ ] Настройка МОС (минимального остаточного срока): `GET/PUT /api/v1/settings/mos` — процент, `admin`-only
- [ ] shadcn/ui: `npx shadcn@latest add progress alert-dialog badge`

## Логика

### FEFO-алгоритм на клиенте

Данные приходят уже отсортированными по `expiry_date ASC` с сервера.  
На клиенте — только **отображение** и **расчёт МОС**:

```ts
const calculateMos = (expiryDate: Date, mosPercent: number): boolean => {
  const totalLife = expiryDate.getTime() - manufactureDate.getTime()
  const remaining = expiryDate.getTime() - Date.now()
  const remainingPercent = (remaining / totalLife) * 100
  return remainingPercent >= mosPercent  // true = НЕ заблокировано
}
```

### Данные с сервера

```
GET /api/v1/orders/:id
Response: {
  id, type: "regular" | "cito",
  status: "new" | "assembling" | "ready" | "shipped",
  destination: { id, name, address },
  items: Array<{
    id, product_id, mnn, sku,
    requested_qty,
    fefo_recommendations: Array<{
      batch_id, serial_number,
      manufacture_date, expiry_date,
      available_qty,
      remaining_life_percent: number,  // серверный расчёт
      is_mos_blocked: boolean          // серверная проверка
    }>
  }>
}
```

### МОС-блокировка

- Если `is_mos_blocked = true` → серия отображается с `<Alert variant="destructive">` «Заблокировано: срок годности менее X%»
- Кнопка подтверждения для этой серии — задизейблена
- В списке заказов такие позиции подсвечены `<Badge variant="warning">МОС</Badge>`

### Cito! — приоритизация

Заказы с `type: "cito"` отображаются **в начале списка** (сортировка на сервере) и выделяются `<Badge variant="destructive">🚨 Cito!</Badge>`.

### Подтверждение отгрузки

```
POST /api/v1/orders/:id/ship
→ Сервер авто-генерирует ТТН (PDF)
→ GET /api/v1/orders/:id/quality-registry → PDF реестра сертификатов
→ UI показывает ссылки на скачивание обоих PDF
```

## 🔐 Блок для администратора

### Настройка МОС

Страница `/admin/settings`:
```
МОС (минимальный остаточный срок):
[60] % ← Input + [Сохранить]
Применяется ко всем аптекам-получателям.
```
`PUT /api/v1/settings/mos { mos_percent: number }`  
Видно только `admin`.

### Подтверждение отгрузки

Кнопка «Отгрузить» — только для `admin`, `qp`, `warehouse_manager`.  
`storekeeper` и `pharmacist` — могут только **собирать** (подтверждать позиции), но не отгружать.
